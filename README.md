## 🌟 关于 Weave

### 命名由来

**Tuanki** 是插件的初始名称，基于发音、认知度和引流效果的考虑而命名。在正式版本中，我们将插件更名为 **Weave**（编织），这个名字蕴含着我们的核心理念：

- 🧵 **编织知识**：将零散的知识点编织成完整的知识网络
- 📦 **收集卡片**：系统化地收集和管理学习内容
- 🔄 **重组记忆**：通过科学的间隔重复，重组和强化记忆
- 💡 **便于记忆**：让知识更容易被理解、记住和应用

### 插件定位

Weave 的定位非常明确：

> 基于 Obsidian 插件生态，延伸和探索官方的设计理念，以弥补和完善其在学习闭环中记忆环节的不完善性，构建在 Obsidian 中属于你的最佳体验的沉浸式学习空间。

**核心使命**：
- 🔗 **深度集成**：无缝融入 Obsidian 生态系统
- 🎯 **完善闭环**：补齐知识管理到记忆巩固的最后一环
- ✨ **沉浸体验**：打造专属于你的学习空间
- 🚀 **科学高效**：基于 FSRS6 算法的智能学习系统

1. **透明度与信任** - 所有代码开源，接受社区审计，确保安全和质量
2. **遵循规范** - 符合 Obsidian 官方插件市场要求

这种模式类似于 Sublime Text、JetBrains 系列产品，代码可见但需要许可证激活高级功能。

## 📦 安装

### 方式一：Obsidian 社区插件（推荐）

1. 打开 Obsidian 设置
2. 进入「社区插件」→「浏览」
3. 搜索「Tuanki」
4. 点击「安装」→「启用」

### 方式二：手动安装

1. 下载最新的 Release
2. 解压到 Vault 的 `.obsidian/plugins/tuanki/` 目录
3. 重启 Obsidian
4. 在设置中启用插件

### 方式三：从源码构建

```bash
# 克隆仓库
git clone https://github.com/zhuzhige123/obsidian---Tuanki.git
cd obsidian---Tuanki

# 安装依赖
npm install

# 开发模式
npm run dev

# 构建生产版本
npm run build
```

## 📖 快速开始

### 1. 创建卡片

Tuanki 支持多种卡片类型，使用标准的 Markdown 语法创建：

#### 📌 基础问答卡片

使用 `---div---` 分隔问题和答案：

```markdown
## 什么是间隔重复学习？

间隔重复学习是一种学习技术，通过增加复习间隔来提高长期记忆效果。

---div---

### 核心原理

间隔重复基于**遗忘曲线**理论：
- 在即将遗忘时进行复习
- 每次成功回忆后延长复习间隔
- 逐步将知识转入长期记忆

💡 **提示**：这是最有效的学习方法之一
```

**格式说明**：
- 使用 `---div---` 分隔正面（问题）和背面（答案）
- 背面内容可选，可以只有问题
- 支持完整的 Markdown 语法（标题、列表、代码块等）

#### 📌 挖空卡片

使用 `==高亮语法==` 标记需要挖空的内容：

```markdown
在 JavaScript 中，==map()== 方法用于创建新数组，==filter()== 方法用于过滤数组。

示例代码：
\`\`\`javascript
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(x => x * 2);  // [2, 4, 6, 8, 10]
const evens = numbers.filter(x => x % 2 === 0);  // [2, 4]
\`\`\`
```

**兼容 Anki 语法**：

```markdown
React 的核心概念是 {{c1::组件化}} 和 {{c2::状态管理}}。
```

**格式说明**：
- 使用 `==内容==` 标记挖空（推荐，使用 Obsidian 高亮语法）
- 也支持 `{{c1::内容}}` Anki 格式
- 可以有多个挖空

#### 📌 选择题

使用标准格式创建选择题：

```markdown
Q: 间隔重复学习的核心原理是什么？

A) 每天固定时间复习
B) 在即将遗忘时复习 {✓}
C) 随机复习
D) 只复习难题
```

**支持多选题**：

```markdown
Q: 以下哪些是间隔重复软件？

A) Anki {✓}
B) Notion
C) SuperMemo {✓}
D) Obsidian
E) Tuanki {✓}
```

**带解析的选择题**：

```markdown
Q: FSRS 算法相比 SM-2 的主要改进是什么？

A) 更简单的公式
B) 个性化参数优化 {✓}
C) 更短的复习间隔
D) 固定的难度系数

---div---

💡 **解析**：FSRS 算法引入了个性化参数优化，可以根据用户的实际学习数据调整算法参数，提供更精准的复习时间预测。
```

**格式说明**：
- 使用 `Q:` 标识问题
- 使用 `A) B) C) D)` 等标识选项（圆括号）
- 使用 `{✓}` 标记正确答案
- 使用 `---div---` 分隔解析（可选）
- 支持多个正确答案（多选题）

### 2. 开始学习

1. 点击侧边栏的 Tuanki 图标
2. 选择要学习的牌组
3. 点击「开始学习」
4. 根据记忆情况选择难度：
   - ⚡ **Again（重来）** - 完全不记得
   - 🤔 **Hard（困难）** - 勉强记住
   - 👍 **Good（良好）** - 正确回答
   - ✨ **Easy（简单）** - 轻松记住

### 4. 查看统计

在统计页面查看：
- 📊 学习进度和记忆曲线
- 🔥 每日学习热力图
- 📈 卡片难度分布
- ⏰ 复习时间预测

## 📝 常见问题

### Q: 为什么代码开源还要收费？

**A**: Weave 采用"开源代码 + 商业许可证"模式。代码开源是为了透明度和符合 Obsidian 规范，许可证收入用于支持持续开发和维护。这是软件行业的成熟商业模式（如 Sublime Text、JetBrains）。

### Q: 我可以修改代码绕过许可证验证吗？

**A**: 技术上可以，我们不会阻止。但是：
- 破解版无法获得官方技术支持
- 破解版无法接收自动更新
- 购买许可证支持项目发展
- 正版用户享受无缝升级体验

### Q: 数据存储在哪里？

**A**: 所有数据都存储在您的 Obsidian Vault 中，完全本地化，保护您的隐私。

### Q: 如何备份数据？

**A**: 数据存储在 Vault 的 `.obsidian/plugins/tuanki/data.json` 中，您可以随时备份此文件。

### Q: 支持移动端吗？

**A**: 目前主要针对桌面端优化，移动端支持正在开发中。
